TYPE A ‚Äî Catalog Content
- Repeating blocks with the same HTML structure
- Each block has its own heading (h3/h4) and/or link
- Each block is logically independent
- Blocks can be reordered without breaking meaning
- Typical examples: real estate listings, services, projects, articles with "–ü–æ–¥—Ä–æ–±–Ω–µ–µ"

TYPE B ‚Äî Single Informational Content
- One topic, one narrative

- Content cannot be split without losing meaning
- Typical examples: guides, explanations, holiday calendars, informational pages

üîπ STEP 2 ‚Äî Decision Logic
IF content is TYPE A:
  - Split content into separate posts
  - Each repeating block becomes ONE post
  - Preserve all links exactly as-is
  - If an image exists inside a block:
      - Move it to the first paragraph
      - Set it as the featured image
  - Assign posts to the appropriate category
  - If the original content came from a category description:
      - DELETE the original category description to avoid duplication

IF content is TYPE B:
  - DO NOT split into multiple posts
  - NEVER keep long content inside a category description
  - CREATE exactly ONE post from the full content
  - Move the entire cleaned and structured content into this post
  - Use the category name as:
      - Post title
      - Post slug (unless already occupied)
  - Assign the post to the original category
  - In the category description:
      - Either leave it empty
      - Or replace with a short 1‚Äì2 sentence intro ONLY

- DO NOT delete
- DO NOT overwrite
